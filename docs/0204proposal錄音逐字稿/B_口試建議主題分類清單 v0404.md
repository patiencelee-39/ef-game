## 1. 格式與引用規範

### 1.1 APA引用格式重複問題 🔴 高優先

**問題描述：**
- 每個文獻都出現兩次引用（AI版本 + 人工核對版本）
- 包含不必要的頁數標註
- 頁數僅在直接引用（direct quote）時需要

**建議來源：** 吳昱鋒委員

**原文引述：**
> "通篇有一個問題，就是APA的那個citation的format。每一個文獻都重複兩次。前面那個是AI給，後面那個是我去核對。要把AI的請走。頁數其實沒有必要，因為你都不是direct quote。"

**修正步驟：**
1. 全文搜尋重複的引用格式
2. 刪除AI生成的版本
3. 保留人工核對的正確版本
4. 檢查並移除非直接引用的頁數標註
5. 確認直接引用的部分有加雙引號和頁碼

**預估工作量：** 中等（需逐條檢查）

### 1.2 AI用語調整 🔴 高優先

**問題描述：**
- 使用台灣學術寫作中不常見的AI生成用語
- 語句不夠自然流暢

**建議來源：** 吳昱鋒委員

**原文引述：**
> "AI用語就是要小心，稍微調整一下。例如：常見的模式之類的、旨在什麼、具體而言、至關重要的，這個台灣人不會用的用法。"

**需要調整的AI用語範例：**

| AI用語 | 建議替換 |
|--------|---------|
| 旨在 | 目的是、用來 |
| 具體而言 | 具體來說、例如 |
| 至關重要的 | 非常重要的、關鍵的 |
| 常見的模式 | 常見現象、普遍狀況 |
| 鑒於 | 考慮到、因為 |
| 有助於 | 能夠幫助、可以 |
| 顯著提升 | 明顯提高、大幅改善 |

**修正步驟：**
1. 全文搜尋AI常用語
2. 逐句檢查語句是否自然
3. 用更口語化但仍保持學術性的表達替換
4. 請非AI使用者閱讀，確認自然度

**預估工作量：** 高（需全文仔細閱讀）

## 2. 理論基礎與論述

### 2.1 「跳過聽損」的論述錯誤 🔴 高優先 ⚠️ 理論重大問題

**問題描述：**
- 誤解「語言剝奪假說」的意涵
- 不當推論：從「語言剝奪是主因」跳到「可以跳過聽損問題」
- 邏輯上的嚴重錯誤

**建議來源：** 林桂如委員

**原文引述：**
> "第13頁最後這兩句話，『可以跳過聽力受損的問題，直接進行執行功能的訓練』。我們會覺得他談的其實還是語言經驗這件事，缺乏語言。他並不是代表可以跳過聽損這樣一個概念。你如果是提供給聽損的孩子，你也不可能是跳過他的聽損的這件事。"

**錯誤論述（第13頁）：**
> "這代表可以跳過聽力受損的問題，直接進行執行功能的訓練。"

**正確理解：**
- 語言剝奪假說強調：執行功能落後的主因是「缺乏語言經驗」而非「聽力本身」
- 這不代表可以「忽略聽損」
- 而是應該「繞過聽覺管道」，採用「視覺優先」的設計

**建議修正為：**
> "基於語言剝奪假說，本研究採用視覺優先的設計，降低對聽覺管道的依賴，讓聽損學生能在較少聽力負擔的情況下進行執行功能訓練。"

或

> "本研究並非忽略聽損學生的聽力限制，而是透過視覺化設計，繞過聽覺管道的障礙，直接訓練執行功能核心能力。"

**修正步驟：**
1. 找出第13頁的問題段落
2. 重新撰寫，強調「視覺優先」而非「跳過聽損」
3. 檢查全文是否有類似的不當論述
4. 確認語言剝奪假說的正確引用與詮釋

**預估工作量：** 中等（需重寫段落並檢查全文）

## 3. 遊戲化設計

### 3.1 遊戲元素不足 🟡 中優先

**問題描述：**
- 目前系統是「訓練工具」而非「遊戲」
- 缺乏真正的遊戲元素

**建議來源：** 吳昱鋒委員、林桂如委員、王文伶指導教授（共識）

**原文引述：**

**吳昱鋒：**
> "既然提到遊戲，可以讓它看什麼元素進來，讓它更像遊戲。目前覺得是個小遊戲沒錯，但它還沒有像遊戲的。我指的遊戲可能是排行榜、積分、競爭性。"

**吳昱鋒（拔雞毛遊戲例子）：**
> "遊戲還有一個重要的元素，就是它要有終點。一關一關的這樣子...有贏跟有輸，然後會有終點。要給他們一個Goal，就是目標。"

**林桂如：**
> "要有遊戲感，只要有遊戲感他就會想要一直玩。有競爭感就會想要一直玩。"

**目前缺少的遊戲元素：**

| 遊戲元素 | 目前狀況 | 建議增加 | 優先級 |
|---------|---------|---------|--------|
| **明確終點** | ⚠️ 不明確 | 每關有結束畫面、總進度條 | 高 |
| **勝負機制** | ❌ 無 | 通關條件、失敗重來 | 高 |
| **積分系統** | ❌ 無 | 答對得分、累積獎勵 | 中 |
| **排行榜** | ❌ 無 | 個人最佳、班級排行 | 低 |
| **階段目標** | ⚠️ 部分 | 更明確的關卡目標 | 高 |
| **競爭機制** | ❌ 無 | 與自己的紀錄競爭 | 中 |
| **獎勵回饋** | ⚠️ 簡單 | 虛擬獎章、解鎖內容 | 低 |

**學生的初步想法：**
- 客製化角色選擇（鹹蛋超人等）
- 積分/代幣制度
- 排行榜功能

**委員回應：**
- 客製化角色只是「相關性調整」（relevance），不是真正的遊戲元素
- 需要增加「系統性的遊戲機制」

**學生的疑慮：**
- 特殊生可能因競爭而面紅耳赤
- 需考慮情緒調節問題

**回應學生疑慮的設計方向：**
1. **與自己競爭**而非與他人：顯示個人最佳成績
2. **選擇性競爭**：讓使用者/教師選擇是否開啟競爭功能
3. **合作模式**：團隊合作達成目標，而非個人競爭

**修正方向：**
- 在「討論」或「未來發展」章節說明遊戲化的未來方向
- 不一定要現在實作，但要說明設計規劃
- 可以分階段開發：基礎版（目前）→ 遊戲化版（未來）

**修正步驟：**
1. 在討論章節新增「遊戲化元素的未來發展」
2. 說明目前的限制與未來的規劃
3. 列出階段性開發計畫

**預估工作量：** 低（撰寫未來規劃即可，不需立即實作）

### 3.2 遊戲感 vs. 難度適性的辯論 🟡 中優先

**討論焦點：**
- 學生認為：動態評量（適性難度）可以維持興趣
- 委員認為：遊戲感才是關鍵，難度適中不足以維持長期興趣

**原文對話：**

**學生（珮瑄）：**
> "我剛剛的想法是說，怎麼樣可以讓他一直玩有興趣，其實就是動態評量應該是可以處理這個問題。在他的近側發展區的時候，他覺得有點難又不會太難，其實他是會有興趣的。"

**林桂如委員：**
> "我不這麼想。要有遊戲感，只要有遊戲感他就會想要一直玩，不管是哪一種遊戲感。"

**吳昱鋒委員（拔雞毛遊戲例子）：**
> "我上禮拜跟小朋友玩一個遊戲叫做拔雞毛。在那個狀態之下就是大家就是覺得好玩。我覺得你就是想玩，不管你是跟誰玩。"

**兩種觀點的整合：**

| 觀點 | 主張 | 支持理論 | 限制 |
|------|------|---------|------|
| **難度適性** | 維持在近側發展區可維持興趣 | 維高斯基ZPD理論、動態評量 | 可能不足以長期維持動機 |
| **遊戲感** | 遊戲機制（競爭、目標、終點）才是關鍵 | 遊戲化理論、內在動機 | 需要複雜的系統設計 |

**建議整合方式：**
- **難度適性**是基礎，確保訓練有效性
- **遊戲感**是加值，提升持續使用動機
- 兩者並非互斥，而是相輔相成

**修正方向：**
在討論章節說明：
1. 目前系統以「有效性」為優先（難度適性）
2. 未來將加入遊戲化元素提升「趣味性」
3. 兩者結合才能達到最佳效果

**預估工作量：** 低（撰寫說明即可）

## 4. 研究設計與方法

### 4.1 前測機制的說明不足 🟡 中優先

**問題描述：**
- 前測機制「埋在測驗中」但沒有明確說明如何運作
- 缺乏獨立的前測評估

**建議來源：** 吳昱鋒委員

**原文引述：**

**學生的說明：**
- 採用**動態評量**和**試題反應理論（IRT）**的概念
- 前測不是獨立進行，而是「埋在訓練過程中」
- 可以從遊戲數據中提取前測成績

**委員的回應：**
- 這個設計可以接受
- 但需要明確說明如何從data中拉出前測成績
- 需要在方法論中清楚說明

**需要補充的內容：**

1. **前測的操作型定義**
   - 前測資料的定義：前X次的表現
   - 如何判定「穩定基準線」

2. **數據提取方式**
   - 從CSV數據中如何判定前測成績
   - 統計方法（如：前三次平均、最初穩定表現）

3. **動態評量與IRT的關係**
   - 目前是「概念性」還是「已實作」？
   - 指導教授建議「先不要延伸」，那論文中如何定位？

**修正方向：**
1. 在「研究方法」章節明確說明前測的操作方式
2. 說明數據分析如何區分「前測」、「訓練」、「後測」
3. 將動態評量與IRT標示為「未來發展方向」

**修正步驟：**
1. 撰寫「數據收集與分析」小節
2. 說明前測成績的判定標準
3. 提供數據分析的流程圖

**預估工作量：** 中等（需撰寫方法論說明）

2. 提及系統的潛在擴展應用可能性
3. 說明未來研究可以驗證在其他群體的效果

**預估工作量：** 低（一段說明即可）

### 5.2 目標年齡層的確認 🟡 中優先

**問題描述：**
- 委員詢問目標是否為學前兒童（4-6歲）

**建議來源：** 林桂如委員

**原文引述：**
