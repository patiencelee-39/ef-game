# 教師設定指南 — 從零開始部署你自己的 EF Game

這份指南假設你**完全沒有程式背景**。
每一步都會告訴你「在做什麼」和「為什麼要這樣做」，請照順序操作，不要跳步驟。

---

## 完成後你會得到什麼？

- 一個**只屬於你的**遊戲網站（網址像 `https://你取的名字.web.app`）
- 學生的遊戲資料存在**你自己的資料庫**，和原始作者完全獨立、互不影響
- 未來可以自由修改遊戲參數（題數、難度等等）

## 需要的時間

第一次設定：約 40-60 分鐘。
之後要更新：只需 1-2 分鐘。

---

# 第一步：安裝必要工具（只需做一次）

> **這一步在做什麼？**
> 就像你要做菜之前要先買鍋碗瓢盆一樣，我們需要先在電腦上安裝幾個免費工具。
> 安裝完之後，這些工具就會一直在你的電腦裡，以後不需要重裝。

---

### 1-1. 確認你有 Google 帳號

如果你有 Gmail 信箱，你就已經有 Google 帳號了，跳過這步。

如果沒有 → 前往 https://accounts.google.com/signup 免費註冊一個。

---

### 1-2. 註冊 GitHub 帳號

> **GitHub 是什麼？**
> 一個存放程式碼的網站，像是「程式碼的 Google Drive」。
> 你需要在上面複製一份遊戲程式碼到你的帳號底下。

1. 打開 https://github.com/
2. 點右上角「**Sign up**」
3. 輸入 email、密碼、使用者名稱
4. 依照畫面完成 email 驗證
5. 看到 GitHub 首頁就代表註冊成功

---

### 1-3. 安裝 Node.js

> **Node.js 是什麼？**
> 一個讓你的電腦能夠執行「自動化腳本」的工具。
> 我們之後會用一個腳本幫你自動完成設定，這個腳本需要 Node.js 才能跑。
> 你不需要寫任何程式，只是「安裝」就好。

**Mac 的安裝方式：**

1. 打開 https://nodejs.org/
2. 你會看到兩個綠色按鈕，點左邊那個「**LTS**」（代表穩定版）
3. 瀏覽器會下載一個 `.pkg` 檔案
4. 到「下載項目」資料夾，雙擊打開它
5. 跳出安裝視窗，一直點「**繼續**」→「**同意**」→「**安裝**」
6. 可能會要求輸入你的電腦密碼（就是你開機時輸入的那個密碼）
7. 看到「安裝成功」就完成了，關閉安裝視窗

**Windows 的安裝方式：**

1. 打開 https://nodejs.org/
2. 點左邊那個「**LTS**」綠色按鈕
3. 瀏覽器會下載一個 `.msi` 檔案
4. 到「下載」資料夾，雙擊打開它
5. 跳出安裝精靈，一直點「**Next**」→「**Next**」→「**Install**」
6. 可能會彈出「是否允許變更」的確認視窗，點「**是**」
7. 看到「Finish」就完成了

---

### 1-4. 安裝 Git

> **Git 是什麼？**
> 一個用來「下載程式碼」和「上傳更新」的工具。
> 我們需要用它把遊戲程式碼從 GitHub 下載到你的電腦裡。

**Mac：**

Mac 通常已經內建 Git，不需要另外安裝。後面如果碰到問題再處理。

**Windows：**

1. 打開 https://git-scm.com/downloads/win
2. 頁面會自動開始下載（如果沒有，點「Click here to download manually」）
3. 雙擊下載的檔案
4. 安裝過程中的所有選項都**保持預設不要動**，一直點「**Next**」
5. 最後點「**Install**」→「**Finish**」

---

### 1-5. 學會打開「終端機」

> **終端機是什麼？**
> 就是一個讓你用「打字」來操控電腦的視窗。
> 平常你是用滑鼠點圖示來開啟程式，終端機則是用「打字輸入指令」來操作。
> 接下來的步驟都會在終端機裡面完成，你只需要**把指令複製貼上，然後按 Enter** 就好。

**Mac 怎麼打開終端機：**

1. 按鍵盤上的 `Cmd + 空白鍵`（會打開 Spotlight 搜尋）
2. 輸入「**Terminal**」
3. 按 Enter
4. 會彈出一個視窗，裡面有閃爍的游標 — 這就是終端機

**Windows 怎麼打開終端機：**

1. 點左下角的「開始」按鈕
2. 搜尋「**Git Bash**」
3. 點開它
4. 會彈出一個黑色視窗 — 這就是你的終端機

> **小提示：怎麼在終端機貼上文字？**
> - Mac：`Cmd + V`
> - Windows Git Bash：在視窗裡按滑鼠右鍵 →「Paste」，或用 `Shift + Insert`

---

# 第二步：複製遊戲程式碼

> **這一步在做什麼？**
> 把遊戲程式碼從原始作者的 GitHub「複製一份到你的 GitHub」，
> 然後再「下載到你的電腦」。
> 這樣你就有了一份完全屬於你自己的副本，修改不會影響原始版本。

---

### 2-1. Fork（在 GitHub 上複製一份）

> **Fork 是什麼？**
> 就是在 GitHub 上「複製一份副本到你的帳號底下」。
> 原版和你的副本各自獨立，互不影響。

1. 確認你已經登入 GitHub
2. 用瀏覽器打開原始遊戲的 GitHub 頁面（原作者會提供網址給你）
3. 在頁面的右上角，找到「**Fork**」按鈕，點它

   ![Fork 按鈕在頁面右上角，靠近 Star 和 Watch 的位置]

4. 跳出的頁面直接點「**Create fork**」（不需要改任何選項）
5. 等幾秒鐘，頁面會跳轉 — 注意看左上角的名字，變成了「**你的帳號/倉庫名稱**」
6. 這就代表複製成功了！

---

### 2-2. 下載到你的電腦

打開終端機（上一步教過怎麼開），依序輸入以下指令。

**每一行是一個獨立的指令，貼上後按一次 Enter，等它跑完再貼下一行。**

第一行 — 移動到桌面（讓下載的東西放在桌面上，方便找到）：

```bash
cd ~/Desktop
```

> 按 Enter 後不會有任何回應，這是正常的，代表已經切換成功。

第二行 — 下載程式碼：

```bash
git clone https://github.com/你的GitHub帳號/倉庫名稱.git
```

> ⚠️ **你需要修改這行指令！**
> 把 `你的GitHub帳號` 換成你的 GitHub 使用者名稱。
> 把 `倉庫名稱` 換成你 Fork 出來的那個倉庫名稱。
>
> **怎麼找到正確的網址？**
> 回到你 Fork 出來的 GitHub 頁面 → 點綠色的「**Code**」按鈕 → 複製那個 `https://...` 網址。
>
> 例如：`git clone https://github.com/wang-teacher/ef-game.git`

按 Enter 後會看到下載進度，等它跑完（幾秒到幾十秒）。

第三行 — 進入剛下載的資料夾：

```bash
cd 倉庫名稱
```

> 把 `倉庫名稱` 換成實際的資料夾名稱（跟上面一樣）。
> 例如：`cd ef-game`

> 做完這三行後，你的桌面上會多出一個資料夾，裡面就是遊戲的所有程式碼。
> 而你的終端機現在「在那個資料夾裡面」，準備好進行下一步了。

---

# 第三步：在 Firebase 建立你自己的專案

> **這一步在做什麼？**
> Firebase 是 Google 提供的免費雲端服務。我們要在上面建立一個「專案」，
> 它會幫你做三件事：
> 1. **放網站** — 讓你的遊戲有一個公開網址，學生可以用瀏覽器打開玩
> 2. **存資料** — 儲存學生的遊戲成績、排行榜等等
> 3. **管登入** — 讓學生可以匿名玩、或用 Google 帳號登入上傳成績
>
> 這些都是免費的（一般班級使用不會超出免費額度）。

---

### 3-1. 建立 Firebase 專案

1. 打開 https://console.firebase.google.com/
2. 用你的 Google 帳號登入
3. 點「**新增專案**」（或英文版的「Add project」）
4. **專案名稱**：輸入一個英文名字，例如 `efgame-wang` 或 `efgame-taipei-school`
   - 只能用：英文小寫、數字、連字號（`-`）
   - 不能有空格或中文
   - 這個名字之後會出現在你的網址裡，例如 `efgame-wang.web.app`
5. 點「繼續」
6. **Google Analytics** → 把開關**關掉**（不需要這個功能）→ 點「建立專案」
7. 等待大約 30 秒，看到 ✓ 打勾後，點「**繼續**」
8. 你現在在 Firebase 的專案主頁了

---

### 3-2. 啟用登入功能（Authentication）

> **為什麼需要這個？**
> 遊戲需要知道「誰在玩」，才能存成績、顯示排行榜。
> 我們啟用兩種登入方式：「匿名」（學生不需要帳號就能玩）和「Google」（上傳排行榜用）。

1. 看左邊的選單，找到「**Build**」（建構）→ 點「**Authentication**」
2. 點中間的「**Get started**」（開始使用）按鈕
3. 你會看到一個列表叫「Sign-in method」（登入方式）

**啟用「匿名」登入：**

4. 在列表中找到「**Anonymous**」（匿名）→ 點它
5. 把「**Enable**」（啟用）的開關打開（變成藍色）
6. 點「**Save**」（儲存）

**啟用「Google」登入：**

7. 回到列表，找到「**Google**」→ 點它
8. 把「**Enable**」的開關打開
9. 「Project support email」（專案支援電子郵件）→ 選你的 Gmail 地址
10. 點「**Save**」（儲存）

---

### 3-3. 建立即時資料庫（Realtime Database）

> **這個資料庫是幹嘛的？**
> 用來儲存多人模式的「即時」資料，例如：哪些人在房間裡、誰答了什麼題目。
> 「即時」代表資料更新的瞬間，其他人的畫面也會立刻同步。

1. 左邊選單「**Build**」→「**Realtime Database**」
2. 點「**Create Database**」（建立資料庫）
3. 跳出一個視窗問你地區，選「**Singapore (asia-southeast1)**」
   - 這是離台灣最近的伺服器，速度最快
4. 點「**Next**」（下一步）
5. 安全規則選「**Start in locked mode**」（鎖定模式）
   - 不用擔心，之後部署時會自動用我們已經寫好的安全規則覆蓋掉這個
6. 點「**Enable**」（啟用）

---

### 3-4. 建立 Firestore 資料庫

> **Firestore 跟上面的 Realtime Database 有什麼不同？**
> Realtime Database 是給「即時同步」用的（多人模式）。
> Firestore 是給「長期存放」用的（排行榜、成績紀錄、班級資料）。
> 遊戲兩個都需要。

1. 左邊選單「**Build**」→「**Firestore Database**」
2. 點「**Create database**」（建立資料庫）
3. 地區選「**asia-east1 (Taiwan)**」
4. 點「**Next**」
5. 安全規則選「**Start in production mode**」（生產模式）
6. 點「**Create**」（建立）

---

### 3-5. 建立網站應用程式 + 取得金鑰

> **金鑰是什麼？**
> 就是一組「通行證密碼」，讓你的遊戲程式知道要連接到「你的」Firebase 專案，
> 而不是別人的。等一下設定腳本會問你這些值。

1. 點左上角的 **齒輪圖示 ⚙️** →「**Project settings**」（專案設定）
2. 滑到頁面最下面，找到「**Your apps**」（你的應用程式）區塊
3. 你會看到幾個圖示，點「**</>**」那個（代表 Web 網頁應用程式）
4. 跳出視窗：
   - 「App nickname」（暱稱）輸入 `EF Game`
   - 勾選「**Also set up Firebase Hosting for this app**」
   - 點「**Register app**」（註冊應用程式）
5. 下一個畫面會顯示一段程式碼，找到這個部分（在 `firebaseConfig` 裡面）：

```
apiKey: "AIzaSy...(一串英文數字)"
authDomain: "你的專案名.firebaseapp.com"
databaseURL: "https://你的專案名-default-rtdb.asia-southeast1.firebasedatabase.app"
projectId: "你的專案名"
storageBucket: "你的專案名.firebasestorage.app"
messagingSenderId: "一串數字"
appId: "1:一串數字:web:一串英數"
```

6. **先不要關閉這個頁面！** 等一下設定時要複製這些值
7. 如果不小心關了也沒關係：回到「Project settings」→ 滑到最下面「Your apps」就能再看到

---

# 第四步：執行一鍵設定腳本

> **這一步在做什麼？**
> 我們準備好了一個「自動設定程式」，它會幫你完成所有技術設定。
> 你只需要回答它的問題（主要是貼上剛才 Firebase 的那些金鑰值）。

---

### 4-1. 安裝專案需要的工具包

回到你的終端機（應該還在剛才的遊戲資料夾裡面），輸入：

```bash
npm install
```

> **這在做什麼？** 下載遊戲需要的「小工具包」到你的電腦。
> 會跑 1-3 分鐘，畫面上會出現很多文字在滾動，這是正常的。
> 等到跑完，游標又開始閃爍、可以打字了，就代表完成。
>
> 如果看到黃色的 `WARN` 文字，那是「警告」不是「錯誤」，可以忽略。
> 只有紅色的 `ERR!` 才是真的有問題。

---

### 4-2. 啟動設定腳本

輸入：

```bash
npm run setup
```

> **這在做什麼？** 啟動我們寫好的自動設定精靈，它會一步一步問你問題。

腳本啟動後，你會看到一個歡迎畫面，然後開始問你問題。
以下是每個問題的回答方式：

---

**問題 1：Firebase 登入**

腳本會自動打開你的瀏覽器，顯示 Google 登入頁面。

- 選你剛才建立 Firebase 專案用的那個 Google 帳號
- 如果問你「允許存取」，點「允許」
- 登入成功後，瀏覽器會顯示「Firebase CLI Login Successful」
- **回到終端機**，繼續下一步

---

**問題 2：Project ID**

```
請輸入你的 Firebase Project ID
Project ID:
```

輸入你在第三步取的專案名稱（例如 `efgame-wang`），然後按 Enter。

> **怎麼確認 Project ID？**
> 回到 Firebase Console（https://console.firebase.google.com/）
> → 你的專案名稱下面那行灰色小字就是 Project ID。

---

**問題 3-8：Firebase 金鑰（共 6 個值）**

腳本會依序問你 6 個值：

| 腳本顯示 | 你要做什麼 |
|---------|-----------|
| `apiKey:` | 從 Firebase 頁面複製 apiKey 的值，貼上，按 Enter |
| `authDomain:` | 複製 authDomain 的值，貼上，按 Enter |
| `databaseURL:` | 複製 databaseURL 的值，貼上，按 Enter |
| `storageBucket:` | 複製 storageBucket 的值，貼上，按 Enter |
| `messagingSenderId:` | 複製 messagingSenderId 的值，貼上，按 Enter |
| `appId:` | 複製 appId 的值，貼上，按 Enter |

> ⚠️ **注意！複製的時候：**
> - **不要**包含前後的引號（`""`）
> - **不要**包含結尾的逗號（`,`）
> - 只要引號裡面的內容
>
> 例如你在 Firebase 頁面看到的是：
> ```
> apiKey: "AIzaSyBxxxxxxxxxxxxxxx",
> ```
> 你只需要貼上：`AIzaSyBxxxxxxxxxxxxxxx`

---

**等待建置**

填完所有值之後，腳本會自動開始「建置」（把你的金鑰編進遊戲程式裡）。
你會看到一些文字在跑，最後出現：

```
╔══════════════════════════════════════════╗
║   ✅ 設定完成！                          ║
╚══════════════════════════════════════════╝
```

看到這個就代表設定成功了！

---

# 第五步：部署上線

> **這一步在做什麼？**
> 把你電腦裡的遊戲檔案「上傳到 Firebase 伺服器」，讓全世界都能透過網址連到你的遊戲。
> 同時也會上傳資料庫的安全規則（已經幫你寫好了，保護學生資料不被外人亂改）。

---

在終端機輸入：

```bash
firebase deploy
```

> **這在做什麼？** 把遊戲上傳到你的 Firebase 專案。
> 大約需要 1-2 分鐘。

等它跑完，最後幾行會顯示：

```
✅ Deploy complete!

Hosting URL: https://efgame-wang.web.app
```

---

### 🎉 恭喜！你的遊戲上線了！

打開瀏覽器，貼上那個 `https://xxxx.web.app` 網址，你應該能看到遊戲畫面了。

試試看用手機也打開那個網址 — 學生以後就是用這個網址來玩遊戲的。

# 之後：日常操作

## 我想更新到最新版本的遊戲

如果原作者更新了遊戲功能，你想同步：

1. 打開終端機
2. 一行一行輸入：

```bash
cd ~/Desktop/倉庫名稱
```

```bash
git pull
```

```bash
npm run build:firebase
```

```bash
firebase deploy
```

> 第一行：進入遊戲資料夾
> 第二行：從 GitHub 抓取最新的程式碼
> 第三行：重新建置（因為你的金鑰是你自己的，需要重新編進去）
> 第四行：上傳到 Firebase

## 我只是改了遊戲設定（題數、難度等），想更新線上版

```bash
cd ~/Desktop/倉庫名稱
```

```bash
firebase deploy --only hosting
```

> `--only hosting` 代表「只更新網站檔案」，比較快（不到 30 秒）。

# 常見問題

### Q: 終端機顯示「command not found: node」？

代表 Node.js 沒安裝成功，或是安裝後沒有重開終端機。

解決方式：
1. 完全關閉終端機視窗
2. 重新打開一個新的終端機
3. 輸入 `node --version` 按 Enter
4. 如果還是顯示 command not found → 重新到 https://nodejs.org/ 下載安裝

---

### Q: `npm run setup` 的時候打錯字了怎麼辦？

不用擔心，直接重新跑一次 `npm run setup` 就好了。
跑多少次都不會壞掉，它只是重新寫入你輸入的值。

---

### Q: `firebase deploy` 顯示「Error: Failed to get Firebase project」？

代表你的 Firebase 登入過期了。先重新登入：

```bash
firebase login
```

登入完成後再跑一次：

```bash
firebase deploy
```

---

### Q: 部署成功但打開網址是空白頁面？

這通常是瀏覽器的舊快取造成的。

解決方式：
- Mac：按 `Cmd + Shift + R`（強制重新整理）
- Windows：按 `Ctrl + Shift + R`
- 手機：清除瀏覽器快取，或用「無痕模式」打開

### Q: 學生的遊戲資料會跟原作者混在一起嗎？

**絕對不會。**

每個 Firebase 專案是完全隔離的——就像兩間不同的銀行，
即使外觀看起來一樣，裡面的帳戶資料完全獨立。

你的學生資料只存在你自己的 Firebase 專案裡，原作者看不到，其他教師也看不到。

---

### Q: 我需要跑 `firebase init` 嗎？

**不需要。**

所有的設定檔（`firebase.json`、安全規則等）已經準備好在程式碼裡了。
你唯一需要做的就是用 `npm run setup` 告訴程式「要連到哪個 Firebase 專案」。

---

### Q: 費用會很貴嗎？

Firebase 的免費方案（Spark Plan）包含：
- 每月 10 GB 網站傳輸量（約可支撐數千次遊戲）
- 1 GB 資料庫儲存空間
- 5 萬次/天的讀取

對一般班級（30-40 人、每天玩幾次）來說，**完全免費**，不需要綁信用卡。
只有在極端情況（數千人同時在線）才可能超出，那時 Firebase 會通知你。

---

### Q: 我可以讓多個班級共用同一個網站嗎？

可以。一個 Firebase 專案可以給任意多人使用。
如果你有多個班級，不需要每個班各建一個專案，共用同一個就好。

### Q: 遇到問題怎麼辦？

1. 把終端機畫面上的紅色錯誤文字**截圖**
2. 聯繫原作者，附上截圖
