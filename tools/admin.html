⚠️ 如果「刪除」按鈕出現「權限不足」錯誤，怎麼辦？

  2. 用程式碼編輯器打開專案資料夾中的 firestore.rules 檔案

  4. 把引號裡面的內容替換成你剛複製的 UID（保留前後的引號）
  5. 存檔後，在終端機執行：firebase deploy
  6. 回到這個頁面重新整理，完成！

💡 免費額度說明（點擊展開）

Firebase 免費方案（Spark Plan）每日上限：

  • Firestore 讀取：50,000 次/天
  • Firestore 寫入：20,000 次/天
  • Firestore 儲存：1 GB
  • RTDB 儲存：1 GB
  • RTDB 下載：10 GB/月

如何查看實際用量：

  1. 前往 Firebase Console
  2. 選擇你的專案（Fork 的教師請選擇自己建立的專案）
  3. 左側選單 →「用量與帳單」→「用量」
  4. 可查看 Firestore、RTDB、Auth 的即時用量圖表

什麼時候需要擔心？

  • 排行榜文件超過 10,000 筆 → 建議用下方「✂️ 修剪」按鈕手動清理
  • 日常使用（數十位學生）完全不會碰到上限
  • 若 Console 顯示用量接近 80%，請聯繫開發者或升級至 Blaze 方案
