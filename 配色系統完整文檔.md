# 🎨 執行功能訓練遊戲 - 配色系統完整文檔

## 📋 文檔目的

本文檔統整「聽覺障礙學生執行功能訓練遊戲」的完整配色系統，確保：
1. **遊戲網頁**與**口試簡報**配色完全一致
2. 未來更新時可快速參照
3. 保持視覺識別系統的統一性

---

## 🎯 核心設計理念

### **視覺優先原則**
- 利用聽障學生的視覺處理優勢
- 色彩作為主要的資訊傳遞管道
- 每個回合有獨立的色彩識別系統

### **認知負荷考量**
- 高對比度（深色背景+明亮前景）
- 色彩語意清晰（成功=綠、錯誤=紅、警示=黃）
- 避免過度複雜的漸層或圖案

### **可及性設計**
- 符合WCAG 2.1 AA級對比度標準
- 考量色盲友善（不僅依賴顏色，也有形狀/文字/邊框輔助）
- 跨裝置顯示一致性

```css
:root {

```

## 📊 配色應用規則

### **1. 背景色系統**

| 用途 | 色碼 | CSS變數 | 使用情境 |
|------|------|---------|----------|
| 主背景 | `#1a2332` | `--bg-dark` | 整體網頁背景、投影片背景 |
| 遊戲區域背景 | `#34495e` | `--game-area-bg` | 刺激物顯示區域、內容框 |
| 半透明遮罩 | `rgba(255, 255, 255, 0.05)` | - | 卡片背景、浮動框 |

**設計理由**：深色背景在投影時更清晰，減少眼睛疲勞，提高文字可讀性。

---

### **2. 文字色系統**

| 用途 | 色碼 | CSS變數 | 字重 | 使用情境 |
|------|------|---------|------|----------|
| 主要文字 | `#FFFFFF` | `--text-white` | bold | 標題、重點內容 |
| 次要文字 | `#ecf0f1` | `--text-light` | normal | 說明文字、註腳 |
| 強調文字 | `#FFD700` | `--accent-yellow` | bold | 關鍵術語、重點標記 |

**設計理由**：白色文字在深色背景上對比度最高，黃色用於強調時不會過度刺眼。

---

### **3. 功能色系統（回饋機制）**

| 功能 | 色碼 | CSS變數 | 使用情境 | 視覺呈現 |
|------|------|---------|----------|----------|
| ✅ 正確 | `#2ecc71` | `--success-green` | 正確回饋、成功訊息 | 綠色邊框+叮聲 |
| ❌ 錯誤 | `#e74c3c` | `--error-red` | 錯誤回饋、警告訊息 | 紅色邊框+噗聲 |
| ⚠️ 警示 | `#FFD700` | `--accent-yellow` | 提示訊息、注意事項 | 黃色邊框 |
| ℹ️ 資訊 | `#3498db` | `--primary-blue` | 一般資訊、標題 | 藍色標題列 |

**設計理由**：採用國際通用的交通號誌色彩語意（綠=通行、紅=停止、黃=注意）。

### **範例3：邊框顏色條件判斷（第二回合）**
```javascript
// JavaScript動態設定邊框顏色
function updateBorderColor(hasPerson, isNightTime) {
    const gameArea = document.getElementById('game-area');
    
    if (hasPerson) {
        // 有人出現 → 紅色邊框
        gameArea.style.border = '6px solid var(--person-border)';
        gameArea.style.boxShadow = '0 0 20px rgba(255, 107, 107, 0.5)';
    } else if (isNightTime) {
        // 晚上場景 → 橙色邊框
        gameArea.style.border = '6px solid var(--night-border)';
        gameArea.style.boxShadow = '0 0 30px rgba(243, 156, 18, 0.4)';
    } else {
        // 預設 → 當前回合主題色
        gameArea.style.border = '4px solid var(--current-round-color)';
        gameArea.style.boxShadow = 'none';
    }
}
```

## 🔄 更新流程（重要！）

### **當需要調整配色時**

#### **步驟1：確認修改原因**
- [ ] 使用者測試回饋（如：色彩不夠明顯）
- [ ] 無障礙檢測問題（如：對比度不足）
- [ ] 新增功能需求（如：新增第五回合）
- [ ] 視覺識別優化（如：品牌調整）

#### **步驟2：修改CSS變數**
```css
/* 在網頁的 <style> 區塊找到 :root */
:root {
    --需要修改的變數名: 新的色碼;
}
```

#### **步驟3：同步更新簡報**
1. 開啟 `presentation_complete.html`
2. 搜尋相同的CSS變數名
3. 修改為相同的色碼
4. 測試所有投影片顯示

#### **步驟4：更新本文檔**
- [ ] 更新配色表
- [ ] 更新應用範例
- [ ] 重新檢查對比度
- [ ] 記錄修改原因與日期

#### **步驟5：版本控制**
```bash
git add .
git commit -m "更新配色系統：[簡述修改內容]"
git push origin main
```


```css
--round1-orange: #ff7f00;  /* 更鮮豔的橘色 */

```

⚠️ **注意**：任何調整都必須重新檢查對比度！
